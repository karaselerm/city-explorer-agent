# Governance — CityExplorer Agent (PoC)

## 1. Risk Register

| Риск | Вероятность | Влияние | Детект | Защита | Остаточный риск |
|---|---:|---:|---|---|---|
| Prompt/Tool injection через пользовательский ввод | Средняя | Высокое | Детектор маркеров инъекции + audit log | Валидация входа, deny-list паттернов, tool allowlist | Средний |
| Timeout/недоступность Overpass API | Высокая | Среднее | Рост `tool_timeouts`, ошибки retriever | Ретраи, fallback на cache/sample данные | Низкий |
| Пустая выдача из-за узких фильтров | Средняя | Среднее | `candidate_count < threshold` | Relax constraints + прозрачное warning в ответе | Низкий |
| Нарушение пользовательских ограничений (distance/time) | Низкая | Высокое | Constraint evaluator в конце pipeline | Отсечение маршрута по лимитам, fail-safe fallback | Низкий |
| Утечка персональных данных в логах | Низкая | Высокое | Периодические проверки логов, grep на PII-маркеры | Политика минимизации данных, no precise location, no contacts | Низкий |
| Side effects без подтверждения (`.ics` export) | Низкая | Среднее | Логи отказа/подтверждения export | Explicit confirmation flag `--confirm-side-effects` | Низкий |
| Неконсистентность состояния memory | Низкая | Среднее | Ошибки SQLite, mismatch в истории | Атомарные upsert/insert операции, fallback на пустой профиль | Низкий |

## 2. Политика данных и логов

### Что собирается

- технические метрики: latency, status, fallback usage;
- tool-трейсы: имя инструмента, длительность, тип ошибки;
- агрегированное состояние профиля: предпочитаемые категории, последний город;
- история построенных маршрутов (без PII).

### Что НЕ собирается

- точная геолокация пользователя в реальном времени;
- контакты, документы, платежные данные;
- свободный пользовательский текст целиком (кроме минимально нужных полей запроса).

### Retention

- логи PoC: 14 дней;
- session traces: 7 дней;
- memory профиля: пока не очищено вручную в локальном runtime.

### Форматы

- Logs: `logs/events.jsonl`;
- State: SQLite (`runtime/memory.db`);
- Exports: `outputs/*.md`, `outputs/*.json`, `outputs/*.ics`.

## 3. Guardrails и безопасность исполнения

### Injection защита

- входной запрос проходит проверку на маркеры инъекции;
- внешние данные (Overpass) считаются недоверенными;
- tool layer получает только структурированные параметры;
- запрещены любые shell/tool действия вне описанного контракта.

### Side-effects контроль

- экспорт `.ics` выполняется только при явном подтверждении;
- по умолчанию разрешены только read/compute операции;
- все side-effect операции логируются отдельно.

### Runtime лимиты

- лимит кандидатов retriever: `<=100`;
- timeout tool calls: `12 c`;
- лимит остановок маршрута: `<=6`;
- ограниченный execution flow без рекурсивных циклов.

## 4. Incident response 

1. Обнаружение: рост `tool_errors`/`tool_timeouts` или частые fallback.
2. Изоляция: временное отключение live retriever, переход в sample-mode.
3. Восстановление: проверка API доступности, возвращение live-режима.
4. Пост-мортем: анализ trace/logs, фиксация root cause и обновление risk register.
